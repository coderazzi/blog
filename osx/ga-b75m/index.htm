Mavericks on Gigabyte GA-B75M

I created my first Hackintosh in 2009, and the process required some thinking. There were already quite a few guides, and, of course, some people had done already the dirty work of working out the drivers and the mechanisms to make Os X install on alien, non Apple, hardware, but the process was definitely not simple. I did then in quick succession 3 hackintoshes, using the same approach.

Fast forward to 2012, I built a new computer and hackintoshed it again, this time with Mountain Lion. In the passed 3 years, the hackintoshing landscape had changed very much, and the process to install OsX was as easy as using the tools Unibeast and Mutibeast provided by tonymacx86. The more serious thinking at this stage was the pre-purchase steps, to ensure that the hardware was as Apple-compatible as possible.

Today I decided to upgrade my existing Mountain Lion installation to Mavericks, or, better said, to scrap the Mountain Lion system to have a clean Mavericks one. Again, the process was very simple, and the only difficulty was getting hold of the Mavericks installation application: there are again many ways to achieve this, but I went the easy path, downloading it using a Macbook Pro.

This is the specification of my system:

  • Motherboard: Gigabyte GA-B75M-D3P. With Intel HD4000 graphics.
  • CPU: i7 4770K
  • Memory: 32 Gb
  • Hard Disk: Samsung 840 EVO 240 Gb
  • Extras: IO bracket with USB 3.0 support
  • Case: LianLi PCV-351

The BIOS settings are basically the default ones. In special, VT is enabled, but VT-D must be disabled.

Using UniBeast 4.0.3, Mavericks installation was flawless. Before executing Multibeast, I installed all the upgrades, reboot the machine, and checked what was missing: only audio was not working, time to start Multibeast 6.5.0.

The only issue I found with Multibeast was related to the already installed drivers: even when I only needed to install one specific Audio driver, I still needed to provide the other required drivers. For my machine, it was needed:

  • Quick Start: select DSDT free
  • Drivers:
    • Audio: add ALC887/888b current
    • Disk: add 10.9.4+ TRIM patch ( 3rd party SATA already selected )
    • Graphics: nothing
    • Misc: no changes (FakeSMC already selected)
    • Network: Realtek: add RTL8111
    • System: no changes ( AppleRTC patch already selected )
  • Bootloaders: Chimera (already selected)
  • Customize: No changes, already selected:
    • Basic Boot options
    • Generate CPU states
    • Hibernate mode -desktop
    • Use kernel cache

Afterwards, all working perfectly.
